# Project Structure

This project is organized in the following way:

- **src/**: Contains all the source code for the project.
- **docs/**: Documentation files, including user manuals and API documentation.
- **tests/**: Unit tests and integration tests.
- **README.md**: This file gives an overview of the project and how to use git.

# How to Use Git

1. **Clone the Repository**: To get a local copy of the project, run the following command:
   ```bash
   git clone https://github.com/qburm/Test-Quinten-PRIV.git
   ```

2. **Create a Branch**: For any new feature or bug fix, create a new branch:
   ```bash
   git checkout -b feature/my-new-feature
   ```

3. **Make Changes**: Edit files, add new features, or fix bugs.

4. **Stage Changes**: Add the changes to the staging area:
   ```bash
   git add .
   ```

5. **Commit Changes**: Commit your changes with a descriptive message:
   ```bash
   git commit -m "Descriptive message explaining the changes"
   ```

6. **Push Changes**: Push your changes to GitHub:
   ```bash
   git push origin feature/my-new-feature
   ```

7. **Create a Pull Request**: Go to the GitHub repository and create a pull request for your changes to be reviewed and merged into the main branch.
